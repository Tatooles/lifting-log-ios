GitHub issue 14 / Phase 7 / App Store submission pack

LiftingLog App Store Submission Plan

The issue should create the submission materials and release evidence needed to move LiftingLog toward App Review. It also creates the live support and privacy pages that App Store Connect and the app will point to.

Support site in this repo Plain HTML and CSS Deploy to support.liftinglog.app Icon split to separate ticket

Quick Read

Primary outcome

A repo-tracked App Store submission pack plus live support and privacy URLs.

Small app change

Replace the disabled Privacy and Support rows with release URLs in the app configuration.

Not included

Final icon creation and full TestFlight hardening stay outside this issue.

Execution Flow

Create the static support site

Add SupportSite/ with a support page and privacy policy. Keep it static, readable, and free of analytics, cookies, and backend dependencies.

Deploy it separately on Vercel

Create a separate Vercel project using SupportSite/ as the root directory. Attach support.liftinglog.app.

Wire live URLs into the app

Configure Release to use https://support.liftinglog.app/ and https://support.liftinglog.app/privacy.

Write the App Store submission pack

Add the metadata, App Privacy worksheet, review notes, screenshot plan, production verification checklist, and placeholder audit.

Verify production readiness

Confirm Release points at production Clerk and Convex, links open from Settings, and a disposable production account can sync a completed workout.

Deliverables

Support Site

  • SupportSite/index.html for support and troubleshooting.
  • SupportSite/privacy/index.html for the privacy policy.
  • Shared CSS, responsive layout, no JavaScript requirement.
  • Published at support.liftinglog.app.

App Store Pack

  • App metadata and field-limit-aware copy.
  • App Privacy worksheet for Clerk, Convex, local workout data, and support contact data.
  • Review notes for local logging, auth, sync, export, and account deletion.
  • Screenshot plan and submission checklist.

App Configuration

  • Release builds use live support and privacy URLs.
  • Tests keep placeholder behavior intentional where needed.
  • No Settings redesign beyond replacing release placeholders.

Release Evidence

  • Support and privacy URLs are reachable.
  • Release config uses production identity and endpoints.
  • Production account sign-in and workout sync are smoke tested.

Scope Boundaries

In Scope

  • Static support and privacy pages in this repo.
  • Separate Vercel deployment for support.liftinglog.app.
  • App Store metadata, privacy worksheet, review notes, screenshot plan, and verification checklist.
  • Release URL wiring in the app.

Out of Scope

  • Creating the final app icon.
  • Full issue 15 TestFlight hardening.
  • Analytics, advertising, HealthKit, subscriptions, or broader feature work.
  • Moving or changing the existing app deployed at liftinglog.app.

Done Means

  • Support page opens publicly.
  • Privacy policy opens publicly.
  • Settings links open the live pages.
  • Release build points at production Clerk.
  • Release build points at production Convex.
  • App Store pack is complete enough to fill App Store Connect.
  • Production account smoke test is complete or blocker is documented.
  • App icon dependency is tracked in a separate ticket.

Blockers

Issue 14 should not be considered submission-ready if the support site is not deployed, the URLs are unreachable, Release points at development services, production auth or sync verification fails, or the final icon dependency remains unresolved at submission time.
